# \# Fluxo Brain

# 

# \## Visão

# 

# O Fluxo existe para ajudar pessoas a organizarem suas vidas de forma leve, clara e sustentável.

# 

# O app não deve gerar ansiedade, culpa ou pressão excessiva.

# 

# O objetivo é criar sensação de clareza mental, continuidade e progresso saudável.

# 

# Fluxo deve parecer:

# \- uma mente organizada

# \- um espaço calmo

# \- um assistente confiável

# \- uma extensão da cabeça do usuário

# 

# \---

# 

# \## Filosofia do Produto

# 

# \- Clareza > estímulo

# \- Simplicidade > excesso de features

# \- Continuidade > produtividade extrema

# \- Calma > dopamina barata

# \- Organização deve reduzir ansiedade

# \- O app nunca deve parecer trabalho

# \- Poucas decisões por tela

# \- O usuário deve sentir controle, não cobrança

# 

# \---

# 

# \## O Que Evitar

# 

# \- Overload cognitivo

# \- Gamificação agressiva

# \- Muitas abas e menus

# \- Visual poluído

# \- Excesso de notificações

# \- Scores que geram culpa

# \- Interfaces densas

# \- Complexidade desnecessária

# \- Sensação de atraso constante

# 

# \---

# 

# \## Comportamento Esperado do App

# 

# \- O app deve ser rápido e fluido

# \- A interface deve parecer leve

# \- O usuário deve entender a próxima ação rapidamente

# \- Toda feature deve ter objetivo claro

# \- Feedback visual deve ser discreto mas eficiente

# \- O sistema deve priorizar foco e clareza

# \- O app deve incentivar retorno saudável

# 

# \---

# 

# \## Regras Absolutas

# 

# \- Nenhuma feature entra sem motivo claro

# \- Toda nova funcionalidade deve reduzir fricção ou aumentar clareza

# \- Se algo adiciona complexidade sem ganho real, deve ser removido

# \- O Fluxo não deve copiar apps viciantes ou caóticos

# \- O usuário deve sentir progresso sem pressão psicológica

#
