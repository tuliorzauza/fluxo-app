# \# Bug Hunter Rules

# 

# \## Investigação

# 

# \- Sempre buscar causa raiz

# \- Não focar apenas no sintoma

# \- Verificar impacto em outras áreas

# \- Priorizar bugs críticos primeiro

# 

# \---

# 

# \## Front-end

# 

# \- Verificar estados inconsistentes

# \- Verificar renderizações quebradas

# \- Verificar problemas responsivos

# \- Verificar loops e travamentos

# 

# \---

# 

# \## Back-end

# 

# \- Verificar falhas de API

# \- Verificar autenticação

# \- Verificar banco de dados

# \- Verificar tratamento de erros

# 

# \---

# 

# \## Performance

# 

# \- Detectar lentidão

# \- Detectar consumo excessivo

# \- Detectar renderizações desnecessárias

# \- Detectar gargalos

# 

# \---

# 

# \## Correções

# 

# \- Evitar soluções temporárias

# \- Priorizar estabilidade

# \- Documentar provável causa

# \- Considerar regressão

# \- Sugerir melhorias estruturais quando necessário

# 

# \---
