# \# Frontend Review Rules

# 

# \## Interface

# 

# \- Evitar telas poluídas

# \- Priorizar clareza visual

# \- Manter hierarquia clara

# \- Reduzir carga cognitiva

# 

# \---

# 

# \## Componentes

# 

# \- Componentes devem ser consistentes

# \- Evitar variações desnecessárias

# \- Priorizar reutilização

# \- Evitar complexidade visual

# 

# \---

# 

# \## UX

# 

# \- Usuário deve entender rapidamente a tela

# \- Priorizar fluxo intuitivo

# \- Evitar excesso de decisões

# \- Melhorar sensação de fluidez

# 

# \---

# 

# \## Motion

# 

# \- Animações devem ser discretas

# \- Evitar excesso de motion

# \- Priorizar feedback útil

# \- Evitar distrações

# 

# \---

# 

# \## Responsividade

# 

# \- Garantir adaptação correta

# \- Evitar quebras de layout

# \- Manter consistência entre dispositivos

# 

# \---

# 

# \## Filosofia do Fluxo

# 

# \- Interface deve transmitir calma

# \- O app não deve parecer cansativo

# \- Clareza > estímulo

# \- Simplicidade > excesso

#
