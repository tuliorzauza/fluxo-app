# \# Code Review Rules

# 

# \## Estrutura

# 

# \- Código deve ser modular

# \- Evitar arquivos gigantes

# \- Separar responsabilidades

# \- Evitar duplicação

# 

# \---

# 

# \## Legibilidade

# 

# \- Código deve ser fácil de entender

# \- Nomes devem ser claros

# \- Evitar lógica confusa

# \- Priorizar simplicidade

# 

# \---

# 

# \## Performance

# 

# \- Evitar renderizações desnecessárias

# \- Evitar loops pesados

# \- Evitar carregamentos desnecessários

# \- Detectar gargalos

# 

# \---

# 

# \## Qualidade

# 

# \- Evitar gambiarra

# \- Detectar riscos de regressão

# \- Validar consistência do projeto

# \- Sugerir refatoração quando necessário

# 

# \---

# 

# \## Escalabilidade

# 

# \- Toda implementação deve considerar crescimento futuro

# \- Evitar soluções frágeis

# \- Evitar acoplamento excessivo

# \- Priorizar manutenção futura

# 

# \---
