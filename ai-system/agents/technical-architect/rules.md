# \# Technical Rules

# 

# \## Estrutura

# 

# \- Componentes devem ser reutilizáveis

# \- Evitar lógica gigante em um único arquivo

# \- Separar UI, lógica e dados

# \- Evitar duplicação de código

# \- Priorizar modularidade

# 

# \---

# 

# \## Front-end

# 

# \- Evitar telas poluídas

# \- Evitar re-renderizações desnecessárias

# \- Priorizar responsividade

# \- Manter consistência visual

# \- Componentes devem ter responsabilidade clara

# 

# \---

# 

# \## Back-end

# 

# \- APIs devem ser organizadas e previsíveis

# \- Evitar acoplamento excessivo

# \- Separar regras de negócio

# \- Validar inputs corretamente

# \- Priorizar segurança

# 

# \---

# 

# \## Performance

# 

# \- Evitar dependências pesadas sem necessidade

# \- Minimizar carregamentos desnecessários

# \- Priorizar fluidez

# \- Monitorar gargalos

# 

# \---

# 

# \## Qualidade

# 

# \- Código deve ser legível

# \- Evitar gambiarra

# \- Toda feature deve ser fácil de manter

# \- Toda implementação deve considerar crescimento futuro

# 

# \---
